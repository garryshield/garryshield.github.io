Deploy Hexo to Github by Travis CI

在这篇文章里介绍了怎么在 Github 上通 Hexo 部署一个静态博客。每次修改和新建文章都要通过 hexo generate, hexo deploy 发布到 Github。其中的原理是先生成要上传的文件然后在本地初始化一个仓库再 push 出去。这里介绍怎么通过 Travis CI 持续集成服务，当我们在本地 commit 并 push 是时候自动部署。

3. 设置 Travis CI

首先，访问官方网站 ，点击右上角的个人头像，使用 Github 账户登入 Travis CI。
Travis 会列出 Github 上面你的所有仓库，以及你所属于的组织。此时，选择你需要 Travis 帮你构建的仓库，打开仓库旁边的开关。一旦激活了一个仓库，Travis 会监听这个仓库的所有变化。

4. .travis.yml

Travis 要求项目的根目录下面，必须有一个.travis.yml文件。这是配置文件，指定了 Travis 的行为。该文件必须保存在 Github 仓库里面，一旦代码仓库有新的 Commit，Travis 就会去找这个文件，执行里面的命令。详情见 官方文档。

其中 - secure: <XXX> 中的 <XXX> 是通过 Travis 命令行工具加密后的的 Token。
